Brittany W test scenario: before and after proposed changes

Test profile: Brittany White vs SAP Sr. AI Product Marketing Manager (AI Foundation). The current tool surfaced 8 skills, none of which match the manual analysis of role-essential skills. This document shows what the tool will produce after the proposed changes are implemented.

What will change in the tool

1Rewrite the JD-to-skills extraction so it surfaces role-essence skills, not tooling skills
Current behavior pulls prompting and content-evaluation skills regardless of role. Proposed behavior anchors extraction to the role family (AI Product Marketing in this case), so the top candidates include the communication, product, and governance skills the JD actually emphasizes.
2Always generate 10 candidate skills, then score and rank them
Standard candidate pool of 10 instead of the current variable 8. The top 5 by score become the recommended development set; the remainder are visible but de-emphasized.
3Add the 5-parameter weighted rubric scoring engine
Priority Score = (Importance × 4) + (Breadth × 3) + (Momentum × 3) + (Connectivity × 2) + (Career Signal × 2)
Each of the 10 candidates is scored 1-3 on each parameter. Maximum possible score is 42. Scores and the breakdown are visible to the reviewer, so the recommendation is auditable rather than opaque.
4Split the result into Maintain and Develop groups based on current vs target proficiency
Skills where the user is already at or above the required level are presented as Maintain with no chapter generated. Skills where there is a real proficiency gap are presented as Develop #N in priority order. This is the core insight from the rerun: Brittany has only 2 actual gaps, not 5.
5Enforce ontology-based diversity (max 2 Develop skills per parent node)
If the top-scoring Develop skills cluster in one parent node (for example, three from D.PRD), the third is replaced with the next-highest scorer from a different parent. Prevents skill-cluster fatigue in the learning path.
6New skills page UI with the score breakdown visible
Per your May 15 mockup: a single table with columns for each rubric parameter, a total score column, and the Maintain or Develop status column. Two grouped sections within the table.
Separate fix being tracked
Profile parsing accuracy. From your March 18 note: Brittany's stored profile had her current position as SAP, which is incorrect. The PDF parser will be retested against all 5 of your prep profiles (Brittany W, Jennifer C, Dorothy F, Halyna M, Srushti M) before the next demo round.
