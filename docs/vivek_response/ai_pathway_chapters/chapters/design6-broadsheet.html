Chapter Two
What Makes an
Agent Different
Three ideas that distinguish AI agents from the chatbots and simple AI tools your organisation already uses.
I
Agents Act, Not Just Answer
A regular AI tool responds to a question. An agent takes a sequence of actions — searching databases, sending notifications, updating records — to complete a goal autonomously. The key shift: from answering to doing.
II
Agents Use Tools
An agent has access to defined tools — a calendar system, a patient record database, an email service. It decides which tools to use and in what order, based on what it's trying to accomplish. This is what makes agents powerful in operations contexts.
III
Agents Operate Within Guardrails
Effective agents are not autonomous in an unconstrained sense. They operate within defined permissions, escalation paths, and approval gates — especially critical in healthcare, where decisions have clinical consequences.
Simple AI Tool
Responds to a single prompt. Produces text output. No memory, no actions, no tools. You do all the follow-up work manually.
AI Agent
Breaks goals into steps, uses tools to execute them, monitors outcomes, and reports back. You define the goal; the agent handles execution.
Multi-Agent System
Multiple specialised agents collaborate — one handles scheduling, one clinical documentation, one compliance checks — coordinated toward a larger workflow goal.

Two concrete examples drawn from real healthcare operations contexts. Study the scenario, the agent design, and the human oversight requirements.
Example One
Discharge Coordination Agent
Situation
Discharge delays cost your health system an estimated 200 bed-hours per week. The bottleneck is coordination: pharmacy, transport, social work, and family notification happen sequentially rather than in parallel.
📋 EHR discharge trigger
📋 Transport roster
📋 Family contact list
Agent Instruction
Goal: When a discharge order is placed in EHR, coordinate all downstream steps in parallel. Step 1: Notify pharmacy of pending discharge. Set 30-min check-in. Step 2: Reserve transport (soonest available slot + 90 min buffer). Step 3: Alert social work if flag present in record. Step 4: Send family notification SMS using approved template. Step 5: If any step fails, escalate to charge nurse immediately.
Example Two
Staffing Gap Monitor Agent
Situation
Your float pool coordinator spends 40% of their shift manually checking schedules and calling in staff for same-day gaps. An agent can monitor for gaps and initiate outreach, freeing them for higher-value work.
📋 Scheduling system API
📋 Staff availability data
📋 Escalation rules doc
Agent Instruction
Goal: Monitor shift schedule for coverage gaps 4+ hours in advance. Step 1: Check schedule every 60 min for gaps in next 8 hrs. Step 2: Cross-reference float pool availability. Step 3: Send shift offer to top 3 available staff (seniority order). Step 4: If no response in 20 min, escalate to supervisor. Step 5: Log all outreach in staffing system. Never book without human confirmation.
